حافظه stack: این حافظه برای ذخیره سازی موقتی متغیرها و توابع در زمان اجرا استفاده می شود. هنگام ایجاد یک تابع، یک بخش جدید از حافظه stack برای آن تابع ایجاد می شود و هنگامی که تابع اجرا شده و خاتمه یابد، این بحش از حافظه stack خالی می شود و از حافظه آزاد می شود. حافظه stack به صورت لایفو خالی شده و می توان از آن استفاده کرد.

حافظه heap: حافظه heap یک حافظه داینامیک است که برای ذخیره سازی داده هایی استفاده می شود که طول عمرشان تا زمانی که دسترسی به آنها نیاز داریم، ادامه دارد. برخلاف حافظه stack، ایجاد و رها کردن فضای حافظه در حافظه heap به صورت دستی انجام می شود و برنامه نویس مسئولیت مدیریت این حافظه را بر عهده دارد. این نوع حافظه برای ذخیره سازی اشیاء و ساختارهای داده پیچیده و بزرگ می تواند مفید باشد.


variable type ای دیتابیس -

int: 4 بایت
long: 8 بایت
float: 4 بایت
double: 8 بایت
decimal: 16 بایت
char: 2 بایت
string: تعداد بایت متغیر وابسته به طول رشته
به عنوان مثال، یک متغیر int در دیتابیس به طور معمول 4 بایت حافظه را تخصیص می دهد. اما برای متغیرهایی مانند رشته (string)، حافظه تخصیص داده شده به استفاده از تعداد بایت متغیر وابسته است.

لازم به ذکر است که مقادیر فوق به عنوان تخمین و توضیحی برای میزان حافظه استفاده شده در ذخیره سازی انواع داده ها در دیتابیس در C# ارائه شده اند و ممکن است بسته به محیط کاری و رفتار ذیل برنامه، این مقادیر تغییر کنند.

refrence type یا variable type -

Enumerations (enum): Enumerations (یا enum) به عنوان یک value type شناخته می‌شوند و در حافظه stack ذخیره می‌شوند.

Records: Records نیز به عنوان یک value type شناخته می‌شوند و در حافظه stack ذخیره می‌شوند.

Structs: Structs همچنین به عنوان یک value type شناخته می‌شوند و در حافظه stack ذخیره می‌شوند.

Tuples: Tuples نیز به عنوان یک value type شناخته می‌شوند و در حافظه stack ذخیره می‌شوند.

بنابراین، Enumerations، Records، Structs و Tuples همگی از نوع value type هستند و در حافظه stack ذخیره می‌شوند.
